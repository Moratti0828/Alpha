### 📘 阶段一：本地 3090 服务器（模型训练 & 打包）

这是你的高性能计算中心，负责把模型算出来。

#### 1\. 环境搭建 (Conda + PyTorch GPU)

```bash
# 1. 创建纯净环境 (Python 3.10 是最稳的)
conda create -n gxb python=3.10 -y

# 2. 激活环境 (每次干活前必做)
conda activate gxb

# 3. 安装 GPU 版 PyTorch (RTX 3090 必须用这个，否则慢死)
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# 4. 安装其他依赖 (FastAPI, Pandas, Scikit-learn 等)
# 确保你当前目录下有 requirements.txt
pip install -r requirements.txt
pip install fastapi uvicorn  # 本地测试API需要
```

#### 2\. 准备字体 (防止乱码)

```bash
# 确保 SimHei.ttf 和代码在同一个文件夹
# 如果没有，可以用 cp 命令复制过去，或者直接上传
ls -l SimHei.ttf  # 检查一下在不在
```

#### 3\. 开始训练 (火力全开)

```bash
# 运行最终版脚本 (自动训练 + 保存.pth + 生成HTML报告)
python run_gpu.py

# 显卡监控 (开个新窗口看 3090 跑得欢不欢)
watch -n 1 nvidia-smi
```

#### 4\. 本地测试 API (确保模型没坏)

```bash
# 1. 确保有 work 目录并把模型放进去了 (run_gpu.py 会自动做，但检查一下)
ls -l work/best_model_binary.pth

# 2. 启动本地 API 服务
uvicorn api_server:app --host 0.0.0.0 --port 8000

# 3. 测试调用 (新窗口)
curl -X POST "http://127.0.0.1:8000/predict/risk" -H "Content-Type: application/json" -d '{"user_id": 888}'
```

### 🚚 阶段二：文件传输（从本地 -\> 腾讯云）

假设你已经在本地生成好了 `best_model_binary.pth` 和写好了 `api_server.py`。

**你需要传输这 3 个文件：**

1.  `api_server.py`
2.  `requirements.txt`
3.  `work/best_model_binary.pth`

*(建议使用 FileZilla / WinSCP 拖拽最方便，如果非要用命令行 scp，参考下面)*

```bash
# 假设腾讯云 IP: 111.230.xx.xx，用户: root，目标目录: /root/gxb_deploy/

# 1. 传代码
scp api_server.py root@111.230.xx.xx:/root/gxb_deploy/
scp requirements.txt root@111.230.xx.xx:/root/gxb_deploy/

# 2. 传模型 (注意要在云端先 mkdir -p work)
scp work/best_model_binary.pth root@111.230.xx.xx:/root/gxb_deploy/work/
```

### ☁️ 阶段三：腾讯云服务器（生产环境部署）

这是你的 API 对外提供服务的地方。

#### 1\. 环境准备 (轻量级 CPU 版)

```bash
# 1. 进入目录
cd /root/gxb_deploy

# 2. 安装 CPU 版 PyTorch (云服务器没显卡，装这个省空间、下载快)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu

# 3. 安装 Web 服务框架
pip install fastapi uvicorn pydantic
```

#### 2\. 文件结构检查 (必做！)

运行 `ll` 或 `tree`，确保结构是这样的：

```text
gxb_deploy/
├── api_server.py
├── requirements.txt
└── work/
    └── best_model_binary.pth  <-- 必须在 work 子文件夹里！
```

*如果 `.pth` 在根目录，记得：`mkdir work && mv best_model_binary.pth work/`*

#### 3\. 启动服务 (两种方式)

**方式 A：前台运行 (调试用，关窗口就断)**

```bash
uvicorn api_server:app --host 0.0.0.0 --port 8000
```

**方式 B：后台静默运行 (正式用，永不掉线) —— ⭐ 推荐**

```bash
nohup uvicorn api_server:app --host 0.0.0.0 --port 8000 > api.log 2>&1 &
```

#### 4\. 运维指令 (生杀大权)

**查看服务是不是在跑：**

```bash
ps -ef | grep uvicorn
# 或者看端口
netstat -tunlp | grep 8000
```

**查看运行日志 (看看有没有报错/有没有人调用)：**

```bash
tail -f api.log
```

**关闭/重启服务：**

```bash
# 1. 找到进程 ID (PID)
ps -ef | grep uvicorn
# 假设看到的 ID 是 12345

# 2. 杀掉进程
kill -9 12345

# 3. 重新启动 (用 nohup 那句)
nohup uvicorn api_server:app --host 0.0.0.0 --port 8000 > api.log 2>&1 &
```
