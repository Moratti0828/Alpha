## 1. 系统概述

**AlphaMind** 是一款基于大语言模型（LLM）的智能投顾与行为感知单页应用（SPA），主要功能包括：

| 模块 | 说明 |
|------|------|
| 投资人格评测 | 9道问卷题，AI 分析生成风险画像与资产配置方案 |
| 行为感知风险模型 | 展示 AI 输出的雷达图、分数、多角色市场观点 |
| 资产配置建议 | 使用后满意度问卷，收集用户反馈 |
| 仪表盘首页 | 综合资产概览、持仓分布、收益趋势可视化 |
| 管理员后台 | 查看与分析所有问卷数据 |

系统采用**纯前端架构**，无需安装任何软件，打开浏览器即可使用；评测数据通过后端 API（`http://111.230.52.99:8000`）进行 AI 分析，问卷数据存储于浏览器本地（localStorage）。

## 4. 功能详解

### 4.1 仪表盘首页

**URL Hash**：`#dashboard`（或直接访问 `index.html`）

#### 页面布局

| 区域 | 内容 |
|------|------|
| 左侧导航栏 | 蓝色渐变背景，分"核心功能"和"系统管理"两组 |
| 顶部状态栏 | 面包屑导航、实时时间、网络/CPU/温度状态 |
| 统计卡片（3个） | 总资产规模、净值增长、风险承受等级 |
| 图表区（3个） | 资产分布饼图、近30日收益趋势、持仓行业分布 |
| 持仓明细表格 | 代码、名称、类型、数量、当前价、收益、收益率、市值 |

#### 风险承受等级卡片

- 默认显示示例数据（稳健型，78分）
- **如果已完成评测**，风险等级卡片会自动从 localStorage 读取最新评测结果，显示真实的风险类型与分数

#### 图表说明

| 图表 | 类型 | 数据来源 |
|------|------|---------|
| 资产分布情况 | 环形饼图 | 示例数据（股票42%、基金28%、债券20%、其他10%） |
| 近30日收益趋势 | 面积折线图 | 动态生成示例数据 |
| 持仓行业分布 | 柱状图 | 示例数据（科技、金融等6个行业） |

> 图表会在窗口大小改变时自动调整尺寸。

### 4.2 投资评测（智能投顾分析）

**URL Hash**：`#assessment`

#### 问卷题目说明

| 题号 | 问题 | 含义 |
|------|------|------|
| 1 | 投资经验有哪些？ | 评估用户的投资知识基础 |
| 2 | 您的工作状态？ | 反映收入稳定性与抗风险能力 |
| 3 | 收入主要来自？ | 判断收入来源的稳定程度 |
| 4 | 家庭可支配年收入？ | 确定可投资资金规模 |
| 5 | 家庭收入有多少钱用于投资？ | 评估投资比例与风险承受能力 |
| 6 | 每月钱花哪了？ | 了解日常支出结构与资金流动性 |
| 7 | 可接受的最长投资期限？ | 判断时间维度上的风险承受度 |
| 8 | 能承受的最大损失？ | 直接量化风险承受能力 |
| 9 | 投资目标是？ | 确定最终投资意图与期望回报 |

#### 如何提交

1. 填写可选的"用户标识"（用于后端日志追踪，可留空）
2. 逐题选择最符合当前情况的选项
3. 点击 **⚡ 开始分析并查看结果**
4. 等待 AI 分析（约 5–15 秒），完成后**自动切换到评测结果页**

> 如果 API 无法访问，会弹出错误提示。请检查网络连接并重试。

#### 调试功能

- 展开"高级（仅调试用）"可查看发送给 AI 的完整提示词（Prompt）

### 4.3 评测结果（行为感知风险模型）

**URL Hash**：`#result`

> 需要先完成投资评测才能查看结果。

#### 结果说明

| 指标 | 说明 |
|------|------|
| **风险类型** | 四档之一：低风险稳健型 / 中等风险平衡型 / 高风险高回报型 / 另类投资偏好 |
| **风险分数** | 0–100 分，分数越高风险承受能力越强 |
| **简评** | AI 生成的 30 字个性化描述 |
| **user_score** | 后端返回的用户综合评分 |

#### 图表说明

| 图表 | 内容 |
|------|------|
| **画像雷达** | 五维雷达图：收益目标、投资经验、资金实力、投资时长、风险承受 |
| **资产建议** | 核心资产 / 卫星资产 / 现金管理的配比建议 |
| **产品能力对比** | 不同产品在多维度的对比雷达图（需 API 返回 comparison 数据） |
| **近7日走势** | 近期走势折线图（需 API 返回 chart 数据） |
| **市场观点** | 基本面分析师、技术交易员、首席风控官、核心辩论、最终决策书 |

#### 操作按钮

- **复制结论**：将风险类型+分数+简评复制到剪贴板
- **导出 JSON**：将完整评测数据下载为 `.json` 文件
- **重新评测**：返回评测问卷，清空当前结果后重填
- **参与使用后问卷**：跳转到满意度调查

### 4.4 使用问卷（资产配置建议）

**URL Hash**：`#survey`

问卷共四个部分，预计需要 3–5 分钟完成。

#### 第一部分：基本信息（必填）

| 字段 | 选项 |
|------|------|
| 性别 | 男 / 女 / 其他 |
| 年龄 | 18–25岁 / 26–35岁 / 36–45岁 / 46–55岁 / 56岁及以上 |
| 教育背景 | 高中及以下 / 专科 / 本科 / 硕士 / 博士 / 其他 |
| 职业领域 | 金融业 / 科技 / 学术 / 公共服务 / 企业运营 / 自由职业 / 学生 / 其他 |
| 投资经验 | 少于1年 / 1–3年 / 3–5年 / 5年以上 |
| 是否使用过智能投顾 | 是 / 否 |

#### 第二部分：Likert 量表（必填，共 15 题）

采用 1–5 分制：
- **1** = 非常不同意
- **2** = 不同意
- **3** = 一般
- **4** = 同意
- **5** = 非常同意

#### 第三部分：开放题（可选填）

1. 最吸引您的功能及原因
2. 对隐私保护、合规性的建议
3. 是否愿意付费及理由

#### 第四部分：联系方式（可选）

用于研究团队后续回访，留空不影响提交。

#### 填写进度

页面顶部有进度条实时显示填写完成度，帮助您了解剩余题目数量。

#### 提交

完成所有必填项后点击**提交问卷**，会弹出感谢弹窗并显示答卷编号，数据保存到本地。

## 5. 常见问题（FAQ）

### Q1：点击"开始分析"后一直转圈，没有结果？

**原因**：后端 API 服务器（`http://111.230.52.99:8000`）无法访问。

**解决方法**：
1. 检查网络连接是否正常
2. 确认 API 服务器是否在运行（可在浏览器地址栏访问 `http://111.230.52.99:8000` 测试）
3. 如果使用的是 HTTPS 页面，可能存在混合内容限制（API 为 HTTP），请改用 HTTP 访问页面

---

### Q2：图表不显示，只看到空白区域？

**原因**：ECharts CDN 无法加载。

**解决方法**：
1. 检查网络是否能访问 `https://cdn.jsdelivr.net`
2. 如无法访问，可下载 ECharts 到本地，修改 HTML 中的 `<script src="...">` 为本地路径

---

### Q3：评测结果页显示"没有找到评测结果"？

**原因**：localStorage 中没有评测数据。

**解决方法**：
1. 点击"前往评测"，完成问卷后会自动跳转到结果页
2. 如果之前完成过评测，可能是清除了浏览器缓存导致数据丢失

---

### Q4：在手机上侧边栏无法打开？

**解决方法**：点击左上角的 **☰（汉堡菜单）** 按钮展开侧边栏。点击内容区域或侧边栏以外的区域可关闭侧边栏。

---

### Q5：问卷提交后数据保存在哪里？

数据保存在浏览器的 **localStorage** 中，键名为 `alphamind_survey_responses_v1`。数据不会自动同步到服务器，仅在当前浏览器可见。管理员可通过管理后台查看。

---

### Q6：如何清除已有的评测数据？

在浏览器开发者工具（F12）的 Application → Local Storage 中，找到对应的键并删除：
- 评测数据：`alphamind_assessment_latest`
- 问卷数据：`alphamind_survey_responses_v1`

## 6. 本地运行方法

> 直接双击 HTML 文件可能存在跨域限制，推荐使用本地服务器。

### 方法一：Python 内置服务器（推荐）

```bash
# 进入 AlphaMind_Web 目录
cd AlphaMind_Web

# Python 3
python -m http.server 8080

# Python 2
python -m SimpleHTTPServer 8080
```

然后在浏览器访问：`http://localhost:8080`

1. 安装 VS Code 扩展：**Live Server**（Ritwick Dey）

3. 浏览器自动打开 `http://127.0.0.1:5500`

```bash

npm install -g http-server

# 在 AlphaMind_Web 目录下运行
http-server -p 8080

# 访问 http://localhost:8080
```

## 7. 技术架构说明

### 文件结构

```
AlphaMind_Web/
├── index.html              # 主单页应用（Dashboard SPA）★ 本文件
├── assessment.html         # 独立评测页（保留兼容）
├── result.html             # 独立结果页（保留兼容）
├── survey.html             # 独立问卷页（保留兼容）
├── admin_login.html        # 独立管理员登录页（保留兼容）
├── admin_dashboard.html    # 管理员后台（独立页面）
├── advice_all.html         # 建议汇总页
├── alphamind_logo.png      # 品牌 Logo
├── logo.png                # 备用 Logo
├── README.txt              # 原始说明文件
└── 使用说明书.md            # 本文件
```

### 数据流

```
用户填写问卷
    │
    ▼
前端构建 Prompt（JavaScript）
    │
    ▼
POST /api/analyze  →  后端 AI 服务（http://111.230.52.99:8000）
    │
    ▼
解析 JSON 响应（从 data.advice 中提取 JSON 块）
    │
    ▼
存储到 localStorage（key: alphamind_assessment_latest）
    │
    ▼
切换到 result 视图，渲染图表
```

### localStorage 键值说明

| 键名 | 类型 | 说明 |
|------|------|------|
| `alphamind_assessment_latest` | JSON Object | 最新一次评测的完整数据 |
| `alphamind_survey_responses_v1` | JSON Array | 所有问卷答卷的数组 |
| `alphamind_admin_token_v1` | String（sessionStorage） | 管理员登录 token，值为 `"1"` |

#### `alphamind_assessment_latest` 结构

```json
{
  "meta": {
    "apiBase": "http://111.230.52.99:8000",
    "userId": "guest",
    "ts": "2024-01-01T12:00:00.000Z"
  },
  "answers": {
    "investment_experience": "stocks",
    "job_status": "stable",
    ...
  },
  "prompt": "（发送给 AI 的完整 Prompt）",
  "raw": { "（后端原始返回 JSON）" },
  "result": {
    "risk_type": "中等风险平衡型",
    "score": "72",
    "description": "...",
    "radar_values": [70, 65, 60, 75, 72],
    "suggestions": [...]
  }
}
```

#### `alphamind_survey_responses_v1` 结构

```json
[
  {
    "meta": { "ts": "...", "ua": "...", "response_id": "R..." },
    "basic": { "gender": "male", "age_group": "26-35", ... },
    "likert": { "scale": "...", "items": [...], "values": [4,5,3,...] },
    "open": { "attract": "...", "privacy": "...", "pay": "..." },
    "contact": { "email": "...", "phone": "..." }
  },
  ...
]
```

### 依赖库

| 库 | 版本 | 用途 | 加载方式 |

```json

```
